          - rect.top는 클릭한 사용자의 y좌표
          - window.innerHeight는 브라우저 높이
          - ex) window.innerHeight가 900이라면 브라우저 높이가 900이고, 여기에 사용자 프로필 높이인 273를 빼고 뷰포트 패딩 10까지 빼서 617이 되는데, 결과적으로 프로필의 top은 617보다 내려가면 안되도록 구성
          - ex) Math.min을 사용하면서 rect.top이 300이라면 (300, 617)이 되고 300이 더 작으므로 300이 사용되며 원래 위치 그대로 유지되고, (750, 617)일 경우 617이 더 작으므로 750에 열리지 않고 617까지만 허용해 사용자 프로필이 열리도록 구성
          - 정리하면 프로필이 잘리기 전에 미리 막는 내용

# 오늘 느낀 점

- 그룹 채팅방 패널에 공지사항, 사용자 목록, 그룹 채팅방 만든 날짜 같은 내용이 보여지는데 이 때 공지사항 내용이 없을 경우 사용자 클릭 시에 프로필이 잘리는 문제없이 보여지지만 공지사항 내용이 길게 작성되거나 사용자 목록이 많아질 경우 사용자를 클릭하면 프로필 아래 부분이 약간 잘리면서 다이렉트 메시지를 보낼 수 없게 되기 때문에 프로필이 보이는 화면 하단에서 약간 띄워진 상태로 열리도록 수정한 결과 해결되어 더이상 잘리는 문제가 발생하지 않는 것을 확인했다. 여러 페이지 컴포넌트 내용을 확인해 주석이 필요한지 확인하고 필요한 내용은 간단하게 추가해주고 프래그먼트가 구성된 내용들은 모두 삭제해 최대한 간결하게 보이도록 수정하면서 이전보다 크게 달라진 것은 없지만 그대로 좀 더 깔끔하게 보여진다고 생각이 된다. 다시 주석 관련 처리를 해줘야 하는데 사용자 관련 내용이나 여러 Zustand 내용이 남아있기 때문에 빠르게 처리한 뒤 배포해줄 예정이다.
