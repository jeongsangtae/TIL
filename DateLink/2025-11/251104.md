- 그룹 채팅방 내의 사용자 목록 온라인 유무 실시간 반영과 관련된 내용 및 이전에 구성한 온라인 친구 목록 실시간 반영과 관련된 내용 정리
  - 참고할 내용인 온라인 친구 목록 구조
    - onlineUsers 생성
      - `const onlineUsers = new Map();`
      - `app.set("onlineUsers", onlineUsers);`
      - 현재 온라인 중인 사용자의 목록을 저장하는 Map
    - registerUser 소켓 이벤트
      - `socket.on("registerUser", async (userId) => { onlineUsers.set(userId, socket.id); ... })`
      - 사용자가 로그인 후 소켓에 연결되면, userId를 등록하고 친구에게 해당 친구가 온라인이라는 이벤트를 보내도록 구성
    - /onlineFriends GET 라우터
      - 클라이언트가 페이지 로드 시, 현재 온라인 중인 친구들만 필터링해서 요청할 수 있도록 구성
    - 전체적인 흐름 구조
      - 서버에서 온라인 사용자 목록을 가진 상태에서, key는 유저 \_id 그리고 값은 소켓 연결 ID를 전역 상태로 가지고 있음
      - 사용자가 로그인하면 소켓에 연결되고 registerUser 이벤트가 발생해 서버에서 해당 사용자가 온라인 상태인걸 기억
      - registerUser 이벤트 안에서 친구 목록을 DB에서 찾고, 친구 목록 중 온라인 상태의 친구에게 실시간으로 알림을 보냄
        - 즉 친구가 온라인이면 해당 친구에게 onlineFriend 이벤트를 보내서 친구 중 한 명이 접속했다는 것을 실시간으로 알려줌
      - "/onlineFriends" GET 라우터에서 요청을 보낸 사용자가 누구인지 판별하고 서버의 onlineUsers Map을 불러옴
      - 친구 목록을 가져와 "onlineUsers.has(friendId)"를 통해 friendId가 Map의 key로 존재하는지 온라인 여부 확인
        - DB 전체 친구 목록 중 현재 서버에 연결되어 있는 친구만 필터링해 클라이언트에 반환
      - 필터링한 친구 목록을 클라이언트에 전달
    - 간단 요약
      - 클라이언트 로그인
      - register 소켓 연결
      - onlineUsers.set을 통해 온라인 사용자 목록에서 온라인 상태의 사용자 소켓 ID를 찾음
      - DB에서 친구 목록을 조회하고 온라인 친구에게 onlineFriend 소켓 이벤트를 통해 로그인한 사용자와 해당 친구 관계 데이터를 전달
      - 프론트엔드에서 온라인 친구 목록 조회 시에, onlineUsers.has를 통해 온라인 친구 필터링
      - 필터링한 온라인 친구 목록을 반환해 출력
  - 그룹 채팅방 사용자 목록에서 구성해야 할 구조
    - 예시 1
      - registerUser 소켓 이벤트 내에 구성을 해야 하나?
        - 해당 이벤트 내에서 그룹 채팅방을 찾고 싶어도, roomId에 대한 정보도 없으며 불러오기도 힘들기 때문에 이 방법은 힘듦
        - 해당 이벤트에서 roomId를 불러오기도 애매한 구조이기 때문에 이 방법은 옳지 못하다고 생각함
    - 예시 2
      - joinRoom 소켓 이벤트 내에 구성을 해야 하나?
        - 해당 이벤트 내에서는 roomId가 있지만, userId가 없어서 "onlineUsers.set" 내용을 통해 온라인 사용자 목록을 찾지 못하기 때문에 다른 방법이 필요함
    - joinRoom 소켓 이벤트 내에서 어떤 방식으로 구성할 것인가?
      - 방법 1
        - userId를 불러오는 것 대신에 그룹 채팅방을 찾고, 해당 채팅방에 참여한 사용자 목록을 가지고 온라인 상태를 찾는 방법
      - 방법 2
        - 사용자가 로그인할 때 userId를 소켓 객체에 직접 저장하고 해당 데이터를 사용하는 방법
    - 정리
      - 생각한 두 가지 방법 모두 테스트해보는 것도 좋다고 생각이 되고, 로그인할 때 굳이 소켓 객체에 따로 저장하지 않으려면 방에 참여할 때 해당 방에 참여한 사용자 목록을 불러와 온라인 사용자 목록과 비교해 찾는 내용이 좀 더 간단해보임
      - 추가로, 찾은 온라인 사용자 목록을 그룹 채팅방 사용자 목록을 불러오는 내용에 함께 포함시켜 전달하도록 구성해 온라인 유무가 실시간 반영되도록 해야 함
      - 그룹 채팅방 온라인 사용자 목록을 따로 조회하는 액션과 라우터가 굳이 필요하지 않고, 방에 참여할 때 사용자 목록을 전달하면서 함께 전달하는 구조면 충분하다고 생각됨

# 오늘 느낀 점

- 아바타 이미지 미리보기 관련 여러 에러를 수정해 잘못된 URL이 들어가면 아바타 색과 닉네임 첫글자가 보여지고, 다시 올바른 URL로 변경했을 때 이미지가 미리보기로 보여지도록 구성해 주었다. 그룹 채팅방 내의 사용자 목록 온라인 유무가 실시간 반영될 수 있도록 로직을 추가하기 전에 온라인 친구 목록이 어떤식으로 구성되어 있고 어떤 구조로 작동하는지 정리해 파악하고, 해당 내용을 참고해 그룹 채팅방 내의 사용자 목록 온라인 유무 관련 내용이 어떤 구조 또는 어떤 방식으로 작동하도록 구성해야 할 지 정리를 해보았는데, 이제 구조와 방식을 좀 파악한 것 같아서 방에 참여할 때 해당 방에 참여한 사용자 목록을 불러오고 온라인 유무를 확인해 해당 정보를 전달하는 구조라면 실시간 반영이 가능할 것 같다는 생각이 들어서 내용을 구성해볼 생각이다.
