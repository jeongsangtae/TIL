# 오늘 느낀 점

- 사용자 프로필 세부 내용에서 모달 데이터를 불러와 사용하는데 이 때 이 내용들이 타입 에러를 발생시키는 문제가 발생해 관련 타입 정의를 구성한 뒤 확인한 결과, 더이상 타입 에러가 발생하지 않는 것을 확인할 수 있었다. 그리고 다이렉트 채팅방 패널 또는 사용자 프로필 세부 내용에서 다른 친구를 클릭하면 해당 다이렉트 채팅방으로 이동되도록 구성을 했었는데, 해당 친구와 관련된 다이렉트 채팅방이 없을 경우 새롭게 채팅방을 생성하는 것이 아닌 홈으로 이동하는 오류가 있기 때문에 해당 내용을 수정해 친구 목록에서 클릭했을 때처럼 채팅방이 존재하면 해당 채팅방으로 이동하고, 없을 경우 새롭게 생성해 이동하도록 내용을 구성해볼 생각이다. 친구 목록에서 친구를 클릭했을 때 동작하는 로직과 완전히 동일하게 구성할 수는 없겠지만 유사하게 구성하면 채팅방으로 단순히 이동하는 것이 아닌, 채팅방이 존재하는지 미리 확인해 존재하면 바로 이동 존재하지 않으면 생성 후 이동이 되는 구조로 구성할 생각이다.
