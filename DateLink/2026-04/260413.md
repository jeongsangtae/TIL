- 알림과 테마 모드 프리셋 문제점 및 해결 방법 정리
  - 알림
    - 문제점
      - 알림 메시지가 보여진 후 놓치면 다시 확인할 수 없는 문제 발생
    - 해결 방법
      - 알림 아이콘을 추가해 알림 메시지가 추가될 때마다 알림 아이콘 옆에 뱃지 형식으로 카운트
      - 현재 위치가 홈 화면일 경우 그룹 채팅방 초대 알림은 카운트되지 않음
      - 현재 위치가 친구 요청 화면에 있을 경우 친구 요청 알림은 카운트되지 않음
      - 현재 위치가 알림 메시지가 전달될 수 있는 채팅방에 있을 경우 채팅 메시지 알림은 카운트되지 않음
        - 알림 메시지가 전달되는 채팅방에 있지 않을 경우엔 카운트
        - 알림 메시지가 전달되지 않은 채팅방에 있을 경우에만 카운트되지 않음
      - 홈 화면과 친구 요청 화면의 주소가 동일하기 때문에 activeTab 이라는 내용을 활용해 구분하고 activeTab이 pendding일 경우 친구 요청 화면에 있는 것으로 처리해 친구 요청 알림이 카운트되지 않도록 구성할 예정
        - 일반적으로 "/me" 경로일 경우 홈 화면으로 처리해 그룹 채팅방 초대 알림이 카운트되지 않도록 구성할 예정
      - 알림 아이콘에 커서를 올리면 그 아래로 확인하지 못한 알림 메시지 목록이 보여지도록 구성할 예정
    - 해결 방법대로 내용을 수정하면 놓친 그룹 채팅방 초대, 친구 요청, 메시지 관련 알림을 모아서 한 번에 확인할 수 있게 될 것으로 예상이 되며, 해당 내용들은 일단 DB에 저장하지 않고 클라이언트 단계에서만 저장해 휘발성으로 구성할 예정
      - DB에 바로 저장하게 될 경우 알림은 계속 쌓이고, 정리 로직도 필요하며, 읽음/안읽음 처리 필요도 하고, 여러 복잡성이 많이 증가하기 때문에 일단 로그인 상태에서만 알림이 쌓이고 확인할 수 있도록 구성하고, 로그아웃 시에 모두 초기화되도록 구성할 예정
  - 테마 모드 프리셋
    - 문제점
      - A 계정 로그인 후 설정한 테마 모드가 B 계정으로 로그인 시에 동일하게 보여지는 문제 발생
    - 해결 방법
      - 테마 모드 변경 시 로컬 스토리지에 저장하며 동시에 DB에도 저장
      - 로그인 시 DB에서 테마 모드 정보를 가져와 적용하고 동시에 로컬 스토리지에 저장
      - 로그인 시 불러오는 테마 모드 정보를 덮어씌우는 구조이기 때문에 로그아웃 시에 테마 모드가 삭제되지 않아도 문제없음
    - 해결 방법대로 내용을 수정하게 될 경우, 계정별로 다른 테마 모드를 저장하고 있게 되며 동일한 브라우저에서 다른 계정으로 로그인해도 브라우저에 저장된 테마 모드가 계정에 저장된 테마 모드 내용으로 인해 덮어씌워지며 변경되는 구조로 작동하게 될 것으로 예상됨

# 오늘 느낀 점

- 줄 높이 관련 내용까지 변수화를 진행해주면서 변수화할만한 내용은 모두 해주었다고 생각이 되어, 추가로 수정이 필요한 알림과 테마 모드 관련 내용은 따로 정리를 해주었는데 해당 내용들은 큰 문제는 아니지만 약간 UX가 부족하다는 생각이 들어서 추가로 수정할 생각이다. 이 두 가지 내용 외에도 추가 수정이 필요한 부분들이 조금 있는데 우선 테마 모드 관련 내용부터 처리한 다음 알림 관련 내용을 처리하고 그 다음 사용자 프로필과 사용자 프로필 세부 내용에서 사용중인 아바타와 온라인 점 관련 내용을 재사용할 수 있는지 확인해 처리해줄 예정이다. 테마 모드 내용은 앞에서 정리한대로 우선 로컬 스토리지에 저장되면서 DB에 함께 저장되도록 구성할 예정이며 로그인 시 DB에서 저장된 내용을 불러와 적용하며 로컬 스토리지에 덮어씌우는 방식으로 구성할 생각이다. 그리고 로그아웃 시에 삭제되는 것을 추가할까 생각했지만 어차피 덮어씌우는 방식이라면 굳이 삭제하지 않아도 될 것 같아서 단순하게 덮어씌우는 구조로 구성해볼 생각이다.
