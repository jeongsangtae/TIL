# 오늘 느낀 점

- react-toastify 라이브러리 관련 CSS 내용을 전역적 CSS 파일에 구성하고 toast 내용을 채팅 관련 액션 함수 내부에 구성한 뒤 테스트한 결과, 모두 문제없이 작동되는 것을 확인했기 때문에 여러 액션 함수와 컴포넌트에 toast를 추가해 alert를 대체할 수 있도록 구성해줄 생각이다. 그리고 react-toastify 라이브러리를 액션 함수 내에 구성하는 것은 GET 같은 내용, 컴포넌트에 구성하는 것은 POST나 PATCH 같은 내용이 되는 것이 정석인데 현재 내 상황에서는 UI 분기가 거의 없고 toast는 alert를 대체하는 단순 알림용이라서 일단 로그인과 회원가입을 제외한 내용은 액션 함수 내부에서 구성할 예정이며 두 내용만 내용을 반환해 컴포넌트에서 처리하도록 구성할 예정이다. 추후에 해당 내용이 추가로 수정될 수도 있긴 하지만 지금은 최대한 빠르게 처리하는 것이 우선이라고 생각되기 때문에 빠르게 구성할 수 있는 방식으로 처리해줄 생각이다.
