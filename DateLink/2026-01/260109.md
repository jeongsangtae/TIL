# 오늘 느낀 점

- 사용자 프로필 입력창 스크롤 UX는 스크롤을 바닥에 가까울 경우 자동으로 아래로 붙도록 구성한 내용으로 인해 첫 줄 이후 약간 위로 올라가는 느낌이 들어서 새롭게 추가한 로직을 주석 처리해 약간의 공간만 잘리는 형태로 유지했고, 다른 사용자 클릭 시 입력창 초기화는 key를 사용해

- 사용자 프로필 입력창 스크롤 UX는 여러 내용을 테스트해봤지만 결국, 스크롤이 바닥에 붙어서 이동하는 로직 또한 약간의 불편함이 있는 것을 확인했기 때문에 초기 상태로 되돌려주었고, 사용자를 클릭해 사용자 프로필을 연 뒤 입력창에 내용을 입력하고 다른 사용자 클릭 시 입력창이 초기화되도록 구성을 해주었는데, 해당 내용은 복잡한 useEffect를 사용하지 않고 key props만 추가해 초기화되도록 구성을 해주었으며, 같은 사용자를 사용자 목록에서 패널 또는 패널에서 사용자 목록 내의 사용자 클릭 시 입력창이 초기화되지 않는 문제가 발생해 origin props 내용을 전달받아 key에 포함시켜 구분되어 초기화되도록 구성을 해주었다. 같은 사용자를 클릭했을 때는 초기화되지 않고 그대로 유지되는 것도 고민을 했으나, 해당 내용은 UX 이득 대비 구성이 많이 복잡하다고 생각이 되었으며, 유지보수 또한 번거로워질 것으로 예상되어 굳이 복잡하게 구성할 필요가 없다는 생각이 들어 조금 단순하게 초기화되는 방향으로 구성을 해주었다. key로 초기화하는 내용은 따로 정리해 놓을 생각이며, 사용자 프로필 세부 내용 CSS와 사용자 프로필 세부 내용에서 다이렉트 채팅방으로 바로 이동하는 내용을 추가할 생각이다.
